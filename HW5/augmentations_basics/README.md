## 2. Конфигурации аугментаций (Задания 1, 2, 4)
Реализован динамический класс `AugmentationPipeline` и три пресета:
1. **Light**: Только базовые геометрические трансформации (`RandomHorizontalFlip`, `RandomRotation(15°)`).
2. **Medium**: Геометрия + цветовые искажения (`ColorJitter`) + кастомный фильтр размытия (`CustomRandomBlur`).
3. **Heavy**: Полный спектр искажений, включая кастомный шум (`CustomSaltPepperNoise`) и кастомный контраст (`CustomRandomBrightnessContrast`).

* **Артефакт**: Сравнение визуальных эффектов на одном изображении сохранено в `results/pipeline_comparison.png`.

## 3. Эксперимент с масштабированием разрешений (Задание 5)
Замер производительности производился на пуле из 100 изображений:

| Разрешение | Время обработки (с) | Изменение памяти (Delta RAM) |
| :--- | :--- | :--- |
| **64x64** | 0.4622s | 3.68 MB |
| **128x128** | 0.5076s | 3.27 MB |
| **224x224** | 0.5503s | 3.36 MB |
| **512x512** | 0.7337s | 12.31 MB |

* **Вывод**: Время обработки растет линейно, в то время как потребление памяти резко увеличивается (почти в 4 раза) при переходе к разрешению 512x512 из-за квадратичного роста количества пикселей в тензорах.
