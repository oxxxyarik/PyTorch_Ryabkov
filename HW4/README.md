# Homework 4: Convolutional Neural Networks (CNN) Analysis

## Project Description
Решение содержит исследование эффективности сверточных архитектур в сравнении с полносвязными сетями на базе датасета MNIST, оценку влияния размеров ядер свертки на извлечение признаков, а также реализацию кастомных структурных блоков (Bottleneck, Channel Attention).

## Structure
* `homework_cnn_vs_fc_comparison.py` — Сравнительный анализ полносвязной сети (FC), базовой CNN и архитектуры со сквозными связями (ResNet).
* `homework_cnn_architecture_analysis.py` — Эксперименты с масштабированием рецептивного поля за счет изменения размеров ядер свертки ($3\times3$, $5\times5$, $7\times7$) с генерацией карт активации первого слоя.
* `homework_custom_layers_experiments.py` — Модульные кастомные компоненты: блок Bottleneck для оптимизации числа каналов и механизм Channel Attention (Squeeze-and-Excitation).

## Experiment Results

### 1. CNN vs FC Performance (Задание 1)
* **FC_Net**: Params: 535,818 | Time: 31.54s | Test Acc: 0.9693 (Наибольшее число параметров при худшей обобщающей способности из-за потери пространственной структуры).
* **Simple_CNN**: Params: 29,066 | Time: 25.20s | Test Acc: 0.9804 (Высокая скорость сходимости благодаря инвариантности к сдвигам).
* **ResNet_CNN**: Params: 24,138 | Time: 67.70s | Test Acc: 0.9841 (Минимальное число параметров и наилучшая точность за счет остаточного обучения).

### 2. Architecture Analysis & Activations (Задание 2)
* **Kernel 3x3**: Выделяет тонкие, четкие локальные границы и контуры цифры.
* **Kernel 5x5**: Захватывает более широкие геометрические переходы, добавляя контекст окрестности пикселей.
* **Kernel 7x7**: Размывает мелкие детали, реагируя на общие глобальные паттерны структуры пятна и макро-форму объекта.

### 3. Custom Layers Consistency (Задание 3)
* Проверка геометрии тензоров завершена успешно. Кастомный блок `BottleneckBlock` с интегрированным `ChannelAttention` корректно сохраняет размерность входа без деградации пространственного разрешения (`Preserves dimensions: [2, 64, 32, 32] -> [2, 64, 32, 32]`).
